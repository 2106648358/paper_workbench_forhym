# 奥运项目评估系统文档体系设计

**日期**: 2026-04-10  
**状态**: 已批准  
**类型**: 文档设计

## 背景

当前 `olympic-evaluation-system` 项目的 README.md 是 Vite 模板的默认说明，未描述项目实际功能。openspec 目录已有详细设计文档，但聚焦于方法论和设计决策。需要创建面向技术评审/开发团队的完整文档体系。

## 目标

- 为技术评审/开发团队提供清晰的项目文档
- 标准化规模（2000字左右），避免过于简略或过度展开
- 文档结构清晰，便于维护和扩展

## 设计方案

### 文档结构

```
olympic-evaluation-system/
├── README.md                    # 项目入口文档
│   ├── 项目简介
│   ├── 核心特性
│   ├── 技术栈
│   ├── 快速开始
│   └── 文档导航
│
├── docs/
│   ├── README.md                # 文档目录总览
│   │
│   ├── ARCHITECTURE.md          # 架构文档
│   │   ├── 系统架构图
│   │   ├── 层次结构说明
│   │   ├── 数据流设计
│   │   └── 状态管理设计
│   │
│   ├── DEVELOPMENT.md           # 开发指南
│   │   ├── 环境配置
│   │   ├── 项目结构
│   │   ├── 组件开发规范
│   │   ├── 状态管理说明
│   │   └── 测试说明
│   │
│   └── API.md                   # API/算法文档
│       ├── AHP 计算模块
│       ├── EWM 计算模块
│       ├── 混合权重计算
│       └── 项目评分与预测
```

### 各文档内容规划

#### README.md（入口文档，500字以内）

| 章节 | 内容 |
|------|------|
| 项目简介 | 2-3 句话描述系统用途 |
| 核心特性 | 4-6 个 bullet 点列关键功能 |
| 技术栈 | 技术选型列表 |
| 快速开始 | 3 步完成本地运行 |
| 文档导航 | 链接到 docs/ 下的详细文档 |

#### ARCHITECTURE.md（架构文档，600字以内）

| 章节 | 内容 |
|------|------|
| 系统架构图 | 四层架构图（展示层/状态层/计算层/数据层） |
| 层次结构说明 | 每层职责简述 |
| 数据流设计 | 主要数据流向 |
| 状态管理设计 | Zustand store 结构 |

#### DEVELOPMENT.md（开发指南，600字以内）

| 章节 | 内容 |
|------|------|
| 环境配置 | Node.js 版本、依赖安装 |
| 项目结构 | 目录结构及职责 |
| 组件开发规范 | 命名、组织方式 |
| 状态管理说明 | Hooks 和 Store 使用方式 |
| 测试说明 | 测试命令和用例编写 |

#### API.md（算法文档，500字以内）

| 章节 | 内容 |
|------|------|
| AHP 计算模块 | 幂迭代法、一致性检验 |
| EWM 计算模块 | 熵权法、信息熵 |
| 混合权重计算 | alpha 混合公式 |
| 项目评分与预测 | 评分公式、概率计算 |

### 与现有 openspec 文档的关系

- 现有 `openspec/changes/archive/2026-03-19-olympic-project-evaluation-system/` 设计文档保留
- 新文档体系聚焦**技术实现层面**，提取关键内容重新组织
- 原 openspec 作为方法论和设计决策的完整记录

## 实现计划

1. 创建 `docs/` 目录结构
2. 编写 `README.md`（项目入口）
3. 编写 `docs/README.md`（文档总览）
4. 编写 `docs/ARCHITECTURE.md`（架构文档）
5. 编写 `docs/DEVELOPMENT.md`（开发指南）
6. 编写 `docs/API.md`（算法文档）
7. 更新项目根目录 `README.md` 指向新文档体系

## 验收标准

- [ ] 所有文档使用简体中文
- [ ] 总字数在 2000-2500 字范围内
- [ ] 文档结构清晰，层次分明
- [ ] 包含必要的代码示例和命令
- [ ] 便于开发者快速上手和扩展
