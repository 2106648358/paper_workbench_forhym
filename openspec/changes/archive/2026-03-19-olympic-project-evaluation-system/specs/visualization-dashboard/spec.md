## ADDED Requirements

### Requirement: 用户可查看 AHP 权重计算过程

系统 SHALL 展示 AHP 层次分析法的完整计算过程，包括判断矩阵、一致性检验结果和最终权重。

#### Scenario: 展示判断矩阵
- **WHEN** 用户进入 AHP 面板
- **THEN** 系统显示 6×6 判断矩阵，包含各指标的两两比较值

#### Scenario: 展示一致性检验
- **WHEN** 系统计算 AHP 权重
- **THEN** 系统显示 λ_max、CI、CR 值，并标明是否通过检验（CR < 0.1）

#### Scenario: 展示 AHP 权重结果
- **WHEN** AHP 计算完成
- **THEN** 系统以柱状图形式展示六个维度的权重，并显示具体数值

---

### Requirement: 用户可查看 EWM 权重计算过程

系统 SHALL 展示熵权法的完整计算过程，包括原始数据、标准化结果、信息熵和熵权。

#### Scenario: 展示原始指标数据
- **WHEN** 用户进入 EWM 面板
- **THEN** 系统显示各项目的六维指标原始数据表格

#### Scenario: 展示信息熵计算结果
- **WHEN** 用户查看 EWM 计算详情
- **THEN** 系统显示各指标的信息熵 H_j、信息效用值和熵权 w_j

#### Scenario: 展示 EWM 权重结果
- **WHEN** EWM 计算完成
- **THEN** 系统以柱状图形式展示六个维度的熵权

---

### Requirement: 用户可调整混合权重参数

系统 SHALL 提供交互式参数调整功能，允许用户修改 α 值并实时查看权重变化。

#### Scenario: 调整 α 参数
- **WHEN** 用户拖动 α 滑块（范围 0-1）
- **THEN** 系统实时更新混合权重 W = α × W_AHP + (1-α) × W_EWM

#### Scenario: 查看权重对比
- **WHEN** 用户查看混合权重面板
- **THEN** 系统并列展示 AHP、EWM、Hybrid 三种权重柱状图

#### Scenario: 灵敏度分析
- **WHEN** 用户点击"灵敏度分析"按钮
- **THEN** 系统显示排名稳定性随 α 变化的曲线图

---

### Requirement: 用户可查看项目评估结果

系统 SHALL 展示各奥运项目的综合评分和排名，支持多维度可视化。

#### Scenario: 展示项目排名
- **WHEN** 用户进入项目评估面板
- **THEN** 系统显示所有项目的综合评分排名表格，包含评分和各维度分项

#### Scenario: 展示六维雷达图
- **WHEN** 用户选择一个或多个项目
- **THEN** 系统显示选中项目的六维指标雷达图，支持多项目叠加对比

#### Scenario: 筛选项目
- **WHEN** 用户选择项目类型筛选（核心/新增/候选/已移除）
- **THEN** 系统仅显示符合类型的项目

#### Scenario: 导出排名数据
- **WHEN** 用户点击"导出 CSV"按钮
- **THEN** 系统下载包含项目排名和评分的 CSV 文件

---

### Requirement: 用户可查看预测推荐结果

系统 SHALL 展示 2032 年奥运项目入选概率预测和改进建议。

#### Scenario: 展示入选概率
- **WHEN** 用户进入预测面板
- **THEN** 系统以横向柱状图展示各项目的入选概率，按高/中/低概率分色

#### Scenario: 展示预测摘要
- **WHEN** 用户查看预测结果
- **THEN** 系统显示高概率入选项目列表和可能被淘汰项目列表

#### Scenario: 查看改进建议
- **WHEN** 用户选中某个项目
- **THEN** 系统显示该项目的改进建议卡片，指出需提升的指标维度

---

### Requirement: 用户可输入新项目进行评估模拟

系统 SHALL 支持用户输入自定义项目的六维指标，模拟评估其入选可能性。

#### Scenario: 打开新项目输入面板
- **WHEN** 用户点击"添加新项目"按钮
- **THEN** 系统显示新项目输入表单

#### Scenario: 输入项目基本信息
- **WHEN** 用户填写项目名称
- **THEN** 系统记录项目名称用于结果展示

#### Scenario: 输入六维指标值
- **WHEN** 用户通过滑块或数值输入设置六个维度的指标值
- **THEN** 系统实时预览雷达图形状

#### Scenario: 计算新项目评分
- **WHEN** 用户点击"计算评分"按钮
- **THEN** 系统计算新项目的综合评分，并将其插入排名列表

#### Scenario: 查看新项目排名
- **WHEN** 新项目计算完成
- **THEN** 系统显示新项目的排名位置、入选概率及与相邻项目的对比

#### Scenario: 保存或删除新项目
- **WHEN** 用户完成新项目评估
- **THEN** 用户可选择"保存到候选列表"或"删除"

---

### Requirement: 用户可进行 What-If 指标模拟分析

系统 SHALL 支持用户模拟调整选中项目的指标值，预测改进效果。

#### Scenario: 打开 What-If 面板
- **WHEN** 用户选中某项目并点击"What-If 模拟"按钮
- **THEN** 系统显示该项目的指标调整面板，包含当前值和可调整范围

#### Scenario: 调整指标值
- **WHEN** 用户拖动某指标的调整滑块
- **THEN** 系统实时更新模拟评分和预测概率

#### Scenario: 查看模拟结果对比
- **WHEN** 用户完成指标调整
- **THEN** 系统显示调整前后的评分、排名、入选概率对比

#### Scenario: 查看模拟建议
- **WHEN** What-If 模拟完成
- **THEN** 系统显示"若提升 X 指标至 Y，可提升排名 N 位，入选概率增加 M%"

#### Scenario: 应用模拟调整
- **WHEN** 用户点击"应用此调整"
- **THEN** 系统将模拟值更新到项目中，重新计算排名

---

### Requirement: 用户可在本地运行系统

系统 SHALL 支持本地开发环境运行，通过 npm run dev 启动。

#### Scenario: 启动开发服务器
- **WHEN** 用户执行 npm run dev
- **THEN** 系统启动本地开发服务器，监听 http://localhost:5173

#### Scenario: 热更新
- **WHEN** 用户修改源代码
- **THEN** 系统自动重新编译并刷新页面

#### Scenario: 构建生产版本
- **WHEN** 用户执行 npm run build
- **THEN** 系统生成优化后的静态文件到 dist/ 目录

---

### Requirement: 系统使用静态 JSON 数据

系统 SHALL 从静态 JSON 文件加载数据，无需后端 API 支持。

#### Scenario: 加载配置数据
- **WHEN** 系统初始化
- **THEN** 系统从 /data/config.json 加载全局配置

#### Scenario: 加载项目数据
- **WHEN** 用户进入项目评估页面
- **THEN** 系统从 /data/projects/*.json 加载项目数据

#### Scenario: 数据加载失败处理
- **WHEN** JSON 文件加载失败
- **THEN** 系统显示错误提示，不影响其他功能使用

---

### Requirement: 系统支持响应式布局

系统 SHALL 适配不同屏幕尺寸，支持桌面和平板设备访问。

#### Scenario: 桌面端访问
- **WHEN** 用户使用桌面浏览器访问（宽度 >= 1024px）
- **THEN** 系统显示完整的多列布局

#### Scenario: 平板端访问
- **WHEN** 用户使用平板访问（宽度 768px - 1023px）
- **THEN** 系统自动调整为单列布局，图表自适应缩放
