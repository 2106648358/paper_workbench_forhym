## Decisions

### D1: 报告结构设计

采用 6 章结构，与论文逻辑一致但侧重展示建模思路：

```
1. 研究概述
   └─ 问题背景、研究目标、方法选择

2. 数学建模过程
   ├─ AHP 层次分析法原理
   ├─ EWM 熵权法原理
   └─ 两层混合模型设计

3. 六维指标体系
   ├─ 指标维度定义
   ├─ 子指标设计
   └─ 量化公式与数据来源

4. 数据处理流程
   ├─ 数据来源分类
   ├─ 数据清洗与标准化
   └─ 数据可获取性分析

5. 初步实验结果
   ├─ 模拟专家权重
   ├─ 一致性检验
   └─ 部分项目评分示例

6. 工作进度与计划
   ├─ 已完成工作
   ├─ 待完成工作
   └─ 时间安排
```

**Rationale**: 此结构突出数学建模和数据处理两条主线，符合导师关注的核心问题。

### D2: 内容来源策略

| 章节 | 内容来源 | 说明 |
|------|----------|------|
| 数学建模 | `mathematical-derivation-draft.md` | 提取核心公式和推导过程 |
| 指标体系 | `indicator-system-design.md` | 展示两层结构和子指标设计 |
| 数据处理 | `data_collection/` 目录 | 说明数据来源和清洗方法 |
| 实验结果 | `simulated-expert-results.md` | 展示模拟数据结果 |

**Rationale**: 复用现有成果，避免重复工作，确保内容一致性。

### D3: 输出格式选择

**选择**: Markdown 格式，支持转换为 PDF

**备选方案**:
- LaTeX Beamer：适合答辩演示，不适合书面报告
- Word/PDF：格式控制好，但不利于版本管理
- Markdown：结构清晰，支持公式，可转换为多格式

**Rationale**: Markdown 便于版本管理，支持数学公式（LaTeX 语法），可通过 Pandoc 转换为 PDF 用于提交。
