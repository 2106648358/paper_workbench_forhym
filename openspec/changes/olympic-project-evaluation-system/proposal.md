## Why

国际奥委会在评估奥运项目入选与淘汰时缺乏系统化、可量化的决策支持工具，导致决策过程透明度不足且易受主观因素影响。本系统基于 2024 年 HiMCM 优秀论文的 AHP-EWM 混合模型，构建科学的奥运项目评估与预测系统，为 IOC 提供数据驱动的决策依据，同时完成毕业论文的系统设计与实现。

## What Changes

- **新增混合权重评估模型**：实现 AHP 层次分析法与熵权法 EWM 的组合权重计算模块，综合主观专家判断与客观数据分布
- **新增六维指标评价体系**：构建流行度、性别平等、可持续性、包容性、创新性、安全性六大维度的量化评估框架
- **新增历史数据验证模块**：收集 2000-2024 年奥运项目数据，验证模型预测准确性并进行参数调优
- **新增 2032 年预测功能**：基于历史趋势与新兴运动发展数据，预测未来奥运项目入选与淘汰可能性
- **新增交互式可视化仪表盘**：开发 Web 应用展示项目评分、权重分布、趋势预测等核心结果
- **新增完整数据管道**：从数据采集、清洗、标准化到特征工程的全流程处理能力

## Capabilities

### New Capabilities

- **data-collection-pipeline**: 奥运历史数据采集模块，包括 IOC 官方数据、各运动联合会统计数据、媒体曝光数据等
- **data-preprocessing**: 数据清洗、缺失值处理、异常值检测、标准化与归一化处理模块
- **ahp-ewm-model**: AHP 层次分析法与熵权法 EWM 混合权重计算核心模块
- **evaluation-metrics-system**: 六维指标（流行度、性别平等、可持续性、包容性、创新性、安全性）量化评估体系
- **prediction-engine**: 基于时间序列与趋势分析的奥运项目预测引擎
- **visualization-dashboard**: 交互式 Web 仪表盘，展示评分结果、权重分布、预测趋势
- **report-generator**: 自动生成评估报告与推荐建议的文档生成模块

## Impact

- **数据源依赖**：IOC 官方网站、各国际运动联合会 (IFs) API、Olympedia 数据库、社交媒体 API（Twitter、Instagram）
- **技术栈**：Python（数据处理与建模）、Scikit-learn/Pandas/Numpy（数据分析）、Flask/Streamlit（Web 应用）、ECharts/D3.js（可视化）
- **计算资源**：模型训练与预测需要中等计算能力，建议配置 8GB+ 内存
- **论文写作**：系统输出将直接支撑毕业论文的核心章节（方法论、实验结果、系统展示）
- **时间规划**：需在论文截止日期前完成数据收集→模型构建→系统开发→结果验证→论文写作全流程
